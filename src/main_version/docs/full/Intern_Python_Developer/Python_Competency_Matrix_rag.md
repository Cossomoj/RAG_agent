# Матрица компетенций стажера Python разработчика

## Уровни развития стажера

### Уровень 1: Начинающий стажер (1-2 месяца)

#### Soft Skills (Гибкие навыки):
1. **Готовность учиться и быстро осваивать новое**
   - Активное изучение Python и Django
   - Участие в обучающих программах
   - Чтение технической документации

2. **Умение задавать вопросы и просить помощь**
   - Формулирование четких вопросов
   - Обращение к ментору при блокерах
   - Использование внутренних ресурсов

3. **Восприятие обратной связи**
   - Конструктивное принятие критики
   - Внесение изменений на основе feedback
   - Постоянное улучшение качества работы

4. **Базовая коммуникация в команде**
   - Участие в daily standups
   - Обмен информацией с коллегами
   - Поддержка командной работы

#### Hard Skills (Технические навыки):

1. **Базовое знание Python**
   - Синтаксис языка
   - Типы данных (int, float, str, list, dict, tuple)
   - Управляющие конструкции (if, for, while)
   - Функции и их параметры

2. **Основы ООП**
   - Классы и объекты
   - Наследование
   - Инкапсуляция
   - Полиморфизм

3. **Работа со стандартными библиотеками**
   - `json` - сериализация данных
   - `datetime` - работа с датами
   - `os`, `sys` - системные операции
   - `collections` - специализированные контейнеры

4. **Основы Git**
   - Базовые команды (commit, push, pull)
   - Работа с ветками
   - Создание pull requests

   - Разбивка задач на подзадачи
   - Оценка времени выполнения
   - Приоритизация задач

3. **Инициативность в обучении**
   - Предложение идей по улучшению кода
   - Участие в обсуждениях технических решений
   - Активный вклад в развитие проекта

4. **Коммуникация с командой**
   - Эффективное участие в технических дискуссиях
   - Обмен знаниями с коллегами
   - Помощь другим стажерам

## Критерии оценки прогресса стажера

### Метрики оценки технических навыков:

#### 1. Качество кода (1-5 баллов):
- **1 балл**: Код работает, но не читаем
- **2 балла**: Код читаем, но не оптимален
- **3 балла**: Код следует PEP 8, есть тесты
- **4 балла**: Код хорошо структурирован, покрыт тестами
- **5 баллов**: Код следует best practices, документирован

#### 2. Самостоятельность (1-5 баллов):
- **1 балл**: Постоянно нужна помощь
- **2 балла**: Помощь нужна часто
- **3 балла**: Помощь нужна редко
- **4 балла**: Помощь нужна только в сложных случаях
- **5 баллов**: Полностью самостоятелен

#### 3. Инициативность (1-5 баллов):
- **1 балл**: Выполняет только порученные задачи
- **2 балла**: Иногда предлагает улучшения
- **3 балла**: Регулярно предлагает улучшения
- **4 балла**: Активно участвует в развитии проекта
- **5 баллов**: Лидирует в технических инициативах

# Жизненный цикл разработки ПО (SDLC) для стажера Python разработчика

## Обзор SDLC для стажера

### Что такое SDLC:
SDLC (Software Development Life Cycle) — это стандартный процесс разработки программного обеспечения, который проходит через определенные этапы от получения требований до развертывания.

### Почему важно понимать SDLC стажеру:
Понимание вашей роли в SDLC критически важно для эффективной работы в команде и качественного выполнения задач во время стажировки.

### Стандартные этапы SDLC для стажера:

#### 1. Получение требований
- **Роль**: Product Owner (PO)
- **Действие**: Предоставление неструктурированных требований
- **Формат**: Диалог, описание потребностей
- **Получатель**: Бизнес-аналитик (БА)
- **Роль стажера**: Слушать и понимать требования

#### 2. Формализация требований
- **Роль**: Бизнес-аналитик (БА)
- **Действие**: Формализация требований в структурированном виде
- **Формат**: User Stories (US), Use Cases (UC), технические задания
- **Получатель**: Архитектор
- **Роль стажера**: Изучать и понимать User Stories

from django.db import models
from django.contrib.auth.models import AbstractUser

class User(AbstractUser):
    email = models.EmailField(unique=True)
    phone = models.CharField(max_length=15, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        db_table = 'users'
        verbose_name = 'Пользователь'
        verbose_name_plural = 'Пользователи'

        return self.email

#### 6. Тестирование
- **Роль**: Тестировщик + Python разработчик (стажер)
- **Входные артефакты**:
  - Разработанное решение от разработчика
  - Требования от БА и СА
- **Действие**: Проверка соответствия реализованного запроектированному
- **Виды тестирования для стажера**:
  - Unit тестирование (обязательно для стажера)
  - Integration тестирование (участие)
  - System тестирование (наблюдение)

#### Пример unit тестирования для стажера:
```python
import pytest
from django.test import TestCase
from .models import User

class UserModelTest(TestCase):
    def setUp(self):
        self.user_data = {
            'username': 'testuser',
            'email': 'test@example.com',
            'password': 'testpass123'
        }
    
    def test_create_user(self):
        user = User.objects.create_user(**self.user_data)
        self.assertEqual(user.email, 'test@example.com')
        self.assertTrue(user.check_password('testpass123'))
    
    def test_user_str_representation(self):
        user = User.objects.create_user(**self.user_data)
        self.assertEqual(str(user), 'test@example.com')
```

```python
def calculate_user_statistics(user_id: int) -> dict:
    """
    Рассчитывает статистику пользователя.
    
    Args:
        user_id: ID пользователя
        
    Returns:
        dict: Словарь со статистикой пользователя
        
    Raises:
        UserNotFoundError: Если пользователь не найден
    """
    try:
        user = User.objects.get(id=user_id)
        orders = user.orders.all()
        
        return {
            'total_orders': orders.count(),
            'total_spent': sum(order.amount for order in orders),
            'average_order': orders.aggregate(Avg('amount'))['amount__avg'] or 0
        }
    except User.DoesNotExist:
        raise UserNotFoundError(f"User with id {user_id} not found")
```

# RAG Agent - Улучшенная версия

## 🎯 Обзор улучшений

Эта версия RAG Agent исправляет все обнаруженные проблемы и добавляет новые функции:

### ✅ Исправленные проблемы:

1. **🔧 Пути Docker → Локальные**
   - Убраны жестко прописанные пути `/app/`
   - Настроены относительные пути через config.yaml
   - Добавлены переменные окружения

2. **📦 Недостающие зависимости**
   - Добавлены `requests`, `unstructured`, `pyyaml`
   - Обновлены все `requirements.txt` файлы
   - Версионирование зависимостей

3. **⚙️ Централизованная конфигурация** 
   - Создан `config.yaml` для всех настроек
   - Валидация конфигурации через `validate_config.py`
   - Единые настройки портов, путей, логирования

4. **🌐 Маршрутизация веб-приложения**
   - Добавлен корневой маршрут `/`
   - Обработка статических файлов
   - SPA-совместимые 404 ошибки

### 🚀 Новые функции:

5. **💾 Автоматическое резервное копирование**
   - Скрипт `backup_database.sh`
   - Ротация старых копий
   - Планировщик заданий

6. **🏥 Мониторинг здоровья**
   - Скрипт `health_check.sh`
   - Проверка портов, процессов, HTTP endpoints
   - Анализ логов на ошибки

7. **🔍 Валидация конфигурации**
   - Проверка `config.yaml` перед запуском
   - Создание недостающих директорий
   - Валидация сетевых настроек

## 🚀 Быстрый старт

### 1. Валидация и установка

```bash
# Валидация конфигурации
python3 scripts/validate_config.py

# Установка зависимостей (если нужно)
./install_dependencies.sh
```

### 2. Запуск системы

```bash
# Улучшенный запуск с мониторингом
./run_all_enhanced.sh

# Или стандартный запуск
./run_all.sh
```

### 3. Проверка здоровья

```bash
# Проверка всех сервисов
./scripts/health_check.sh

# Создание резервной копии
./scripts/backup_database.sh
```

## ⚙️ Конфигурация

Все настройки находятся в `config.yaml`:

```yaml
# Сетевые настройки
network:
  rag_service:
    host: "0.0.0.0"
    port: 8000
  webapp_api:
    port: 5001
  admin_panel:
    port: 8002

# Пути к файлам
paths:
  database: "src/main_version/AI_agent.db"
  rag_path: "src/main_version"
  logs_dir: "logs"

# Настройки безопасности
security:
  rate_limiting_enabled: false
  cors_enabled: true
```

## 🔧 Устранение неполадок

### Проблемы с запуском:
1. Проверьте конфигурацию: `python3 scripts/validate_config.py`
2. Проверьте зависимости: `./install_dependencies.sh`
3. Проверьте логи: `tail -f logs/*.log`

### Проблемы с портами:
```bash
# Проверка занятых портов
lsof -i :8000,5001,8002

# Принудительная остановка
./stop_all.sh

# Перезапуск
./run_all_enhanced.sh
```

### Проблемы с базой данных:
```bash
# Восстановление из резервной копии
cp backups/AI_agent_backup_YYYYMMDD_HHMMSS.db src/main_version/AI_agent.db

# Проверка целостности
sqlite3 src/main_version/AI_agent.db "PRAGMA integrity_check;"
```
